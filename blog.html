Local Storage:   The data is stored in local storage for a lifetime unless the user manually deletes it from the browser, the data remains in the browser until and unless the browser's memory is cleared. It has four methods that we can use to set, retrieve, remove and clear i.e setItem(), getItem(), removeItem(), clear()


Session Storage:   The main difference lies in the lifespan as it persists in the browser until its current tab is on. Once you close the tab or terminate it, the data on session storage also gets lost. We can also set and retrieve its data using setItem() and getItem() methods, respectively, similar to the local storage methods.

Global Scope:  The variables defined outside of any function or curly brackets are known as global variables and have global scope. Global scope means that the variables can be accessed from any part of that program, any function or conditional state can access that variable.


Block Scope The block scope can be defined as the scope of the variables inside the curly brackets {}. Now, these curly brackets can be of loops, or conditional statements, or something else. You are only allowed to refer to these variables from within the curly brackets {}.

The Event loop  is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process.

There are 8 ways to get undefined 

  1. If a variable is not inititalized
  2. Function with no return
  3. Parameter that is not passed will be undefined
  4. If return has nothing on the right side will return undefined
  5. Property that doesn't exists on an object will give you undefined
  6. Accessing array elements outside of the index range
  7. Deleting an element inside an array
  8. Set a value directly to undefined
